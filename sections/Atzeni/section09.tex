\section{Network Forensics}

Network forensics involves the capture, recording, and analysis of network traffic to investigate security incidents or legal evidence.\\
Unlike static file system forensics, network forensics deals with volatile data and the consequences of actions performed across digital networks.

\subsection{Conceptual Overview and OSINT Integration}
Modern investigations adopt a multimodal approach, combining traditional packet analysis with \textbf{Open Source Intelligence (OSINT)}.\\
OSINT serves as a supplement to digital forensic data by gathering information from publicly available sources to reconstruct a suspect's digital footprint.

\subsubsection*{Key OSINT Evidence Sources}
\begin{itemize}
    \item \textbf{Archived Snapshots:} Using tools like the \textit{Wayback Machine} to access historical versions of websites that are no longer online.
    \item \textbf{Domain Registration:} Checking the age and ownership of domains. Recently registered domains can often be a "red flag" for malicious campaigns.
    \item \textbf{Public Repositories and Forums:} Analyzing code leaks, forum posts, and usernames to identify affiliations or signs of compromise.
    \item \textbf{Social Media Profiling:} Tracking activity patterns, location data from posts, and sentiment analysis to build a psychological or behavioral profile of a suspect.
\end{itemize}

\subsection{Technical Acquisition and Analysis}
The primary objective in network forensics is the reconstruction of communication timelines. This process is often complicated by encryption (TLS, SSH, IPsec) and obfuscation.

\subsubsection*{The Analysis Workflow}
When deep packet inspection (DPI) is hindered by encryption, investigators must pivot to complementary techniques:
\begin{enumerate}
    \item \textbf{Lower-Layer Analysis:} Accessing metadata at the data link, network, and transport layers to identify IP addresses and connection timestamps.
    \item \textbf{Endpoint Integration:} Accessing the host device to recover symmetric session keys, allowing for the decryption of TLS packets within tools like Wireshark.
    \item \textbf{Pattern Recognition:} Identifying specific traffic behaviors even in the absence of clear-text content.
\end{enumerate}

\subsubsection*{Location and Identity Inference}
Metadata such as the assigned IP address can be used to derive a restricted geographical zone for a physical location. By correlating interaction times and active connections, a network forensic investigator can reconstruct a precise timeline of an individual's digital actions, providing a "gold mine" of clues for sound fact reconstruction.

\subsection{Network Forensic Phases: Preservation to Reporting}

The lifecycle of a network forensic investigation ensures that volatile network data is captured, analyzed for patterns, and documented in a manner that remains defensible in legal contexts.

\subsubsection{Data Preservation and Snapshots}
Because network information is often outside the investigator's direct control (residing on ISP or third-party servers), data can be modified, moved, or deleted at any time.

\subsubsection*{Methodology for Admissibility}
To preserve the "original state" of network evidence for court use, investigators must adopt snapshot-driven tools that:
\begin{itemize}
    \item \textbf{Capture Snapshots:} Generate immediate captures of web archives, social media posts, and public repositories.
    \item \textbf{Digital Signing:} Apply cryptographic signatures and timestamps to verify that the information existed in that specific state at that specific time.
    \item \textbf{Metadata Retrieval:} Replicate associated metadata (IP headers, server logs) to provide a verifiable chain of evidence.
\end{itemize}

\subsubsection{Evidence Analysis and Relationship Mapping}
The analysis phase focuses on identifying relationships between disparate digital artifacts and reconstructing the attack timeline.

\subsubsection*{Conceptual Correlation Graphs}
Analysis typically involves the creation of a graph-based model to identify connections between:
\begin{itemize}
    \item \textbf{Entities:} Relationships between individuals, usernames, and physical locations.
    \item \textbf{Network Artifacts:} Correlation of IP addresses, domain names, and email metadata.
    \item \textbf{Attack Vectors:} Tracing the origin of a digital incident (e.g., distinguishing between a legitimate email and a forged bot-generated spam campaign).
\end{itemize}

\noindent
This process often requires cooperation with third parties, such as network administrators or service providers, to access restricted metadata and identify patterns of malicious behavior.

\subsubsection{Practical Application: IP Geolocation (OSINT)}
A common forensic task is narrowing down the geographical location of a node or user using public databases.

\subsubsection*{Contextual IP Tools}
Tools such as \textbf{IPinfo}, \textbf{ip-api.com}, and \textbf{MaxMind GeoIP} allow investigators to immediately retrieve:
\begin{itemize}
    \item \textbf{Geographical Data:} City, region, and approximate GPS coordinates.
    \item \textbf{Organization Data:} ASN (Autonomous System Number) and the hosting provider or ISP.
    \item \textbf{Security Posture:} Identifying if an IP is associated with a VPN, proxy, or Tor exit node.
\end{itemize}


\noindent
By correlating a suspect's public terminal address with known organizational databases (e.g., Politecnico di Torino), investigators can verify the physical location of a device with high reliability, forming the foundation for further investigation into digital footprints.

\subsubsection{Forensic Reconstruction and Timeline Correlation}
Effective network forensics requires the synchronization of multiple data streams to identify the "ground truth" of a digital event.

\subsubsection*{The Three Pillars of Reconstruction}
\begin{itemize}
    \item \textbf{Causal Correlation:} Establishing a direct link between an action (e.g., a specific packet sent) and its result (e.g., a system crash or unauthorized access).
    \item \textbf{Temporal Synchronization:} Aligning timestamps from varied sources (server logs, ISP metadata, local machine clocks). This often requires accounting for \textbf{Clock Drift} and timezone variations to ensure the sequence of events is accurate.
    \item \textbf{Identity Mapping:} Linking network identifiers (IP addresses, MAC addresses, session tokens) to physical entities or specific user accounts.
\end{itemize}

\subsubsection*{Challenges in Modern Network Analysis}
As networks evolve, forensics must adapt to increasing layers of complexity.

\subsubsection*{The Encryption and Privacy Paradox}
The widespread adoption of end-to-end encryption (E2EE) and VPNs has shifted the focus of network forensics from \textbf{content analysis} to \textbf{traffic analysis}.
\begin{itemize}
    \item \textbf{Traffic Shaping/Fingerprinting:} Identifying an application or OS based on the behavior of the encrypted packets (size, frequency, timing).
    \item \textbf{Log Correlation:} Relying on unencrypted "negotiation" data (such as SNI in TLS handshakes) to determine which domains were contacted, even if the payload is hidden.
\end{itemize}

\subsubsection{Final Reporting: The "So What?" Factor}
The ultimate goal of the investigation is to produce a document that answers the fundamental forensic questions: \textbf{Who, What, Where, When, and Why}.

\subsubsection*{Operational Reporting Guidelines}
\begin{itemize}
    \item \textbf{Objectivity:} The report must strictly adhere to the facts observed in the data, avoiding speculative leaps.
    \item \textbf{Reproducibility:} Another investigator should be able to follow the steps outlined in the report and reach the same conclusions using the same raw evidence.
    \item \textbf{Executive Summary:} A high-level overview that translates technical PCAP data into a narrative understandable by non-technical stakeholders (judges, lawyers, managers).
\end{itemize}

\subsection{Social Media Forensics}

Social Media Forensics involves the identification, preservation, and analysis of data generated on social networking platforms.\\
Given the dynamic nature of these environments, evidence is highly volatile and requires specialized acquisition techniques to maintain its integrity for judicial purposes.

\subsubsection{Data Acquisition and Preservation}
Unlike static local storage, social media data is hosted on third-party infrastructures, making traditional bit-stream imaging impossible. Investigators must rely on a combination of API-based collection and advanced scraping techniques.

\subsubsection*{Methodologies for Legal Admissibility}
\begin{itemize}
    \item \textbf{Remote Snapshot Capture:} Using forensic tools to take authenticated "snapshots" of profiles, posts, and comments. These must include cryptographic hashing and server-side timestamps to prevent claims of manipulation.
    \item \textbf{Metadata Extraction:} Capturing hidden details such as the \textit{User Agent}, original upload timestamps, and associated IP addresses where available.
    \item \textbf{Chain of Custody for Web Evidence:} Documenting the entire path of acquisition, including the URL, the tools used, and the digital signatures applied at the moment of capture.
\end{itemize}

\subsubsection{Forensic Artifacts in Social Media}
The analysis focuses on reconstructing user behavior and identifying the origin of digital content.

\subsubsection{Challenges and Countermeasures}
Modern platforms implement several features that complicate forensic investigations, requiring a pivot toward behavioral and pattern-based analysis.

\subsubsection*{Anti-Forensic Obstacles}
\begin{itemize}
    \item \textbf{Ephemeral Content:} Posts that automatically disappear (e.g., Stories). These require immediate, automated capture triggers.
    \item \textbf{Encryption and Privacy Settings:} End-to-end encryption in messaging (E2EE) prevents middle-man interception, forcing investigators to focus on physical endpoint acquisition (smartphone forensics).
    \item \textbf{Account Deletion and Spoofing:} The ease of creating "sockpuppet" accounts necessitates the use of OSINT to correlate IP addresses and behavioral fingerprints across different sessions.
\end{itemize}

\noindent

\subsubsection*{The Role of OSINT in Attribution}
Social media forensics is rarely performed in isolation. By correlating public profile data with external databases (e.g., domain registration, public code repositories), investigators can transform an anonymous handle into a verifiable identity, providing the "who" behind the digital traces.

\subsection{Anti-forensics Techniques and Countermeasures}

Anti-forensics refers to a set of methodologies and tools designed to frustrate digital forensic investigations by compromising the availability, reliability, or completeness of evidence.\\
The primary objective is to increase the investigator's uncertainty and the resources required for a successful analysis.

\subsubsection{Data Hiding and Obfuscation}
Encryption and steganography represent the two main pillars of data hiding. While encryption makes the content unreadable without a key, steganography aims to hide the very existence of the communication.

\subsubsection*{Steganography in Digital Media}
\begin{itemize}
    \item \textbf{Least Significant Bit (LSB) Insertion:} Altering the redundant bits of an image or audio file to embed secret data. Since the changes are imperceptible to human senses, the file appears legitimate.
    \item \textbf{Detection (Steganalysis):} Investigators use statistical analysis to detect anomalies in the bit distribution of suspected files.
\end{itemize}

\subsubsection*{Encryption and Hidden Volumes}
Advanced anti-forensic tools allow the creation of "hidden volumes" within encrypted partitions. 
\begin{itemize}
    \item \textbf{Plausible Deniability:} The system is designed such that it is mathematically impossible to prove the existence of a second, hidden volume once the first one is decrypted, providing the user with a defense against coerced password disclosure.
\end{itemize}

\subsubsection{Artifact Wiping and Metadata Manipulation}
Unlike simple deletion, wiping ensures that data is physically unrecoverable by overwriting sectors. Metadata manipulation, on the other hand, targets the timeline of events.

\subsubsection*{Timestomping}
Timestomping is the deliberate modification of file system timestamps (Modified, Accessed, Created, and Birth - MACB). 
\begin{itemize}
    \item \textbf{Objective:} To disrupt timeline reconstruction and mislead investigators regarding the sequence of malicious activities.
    \item \textbf{Detection:} Analysts look for inconsistencies between the MFT\\ ($ \$$STANDARD\_INFORMATION vs. $ \$$FILE\_NAME) and system logs or registry keys that may still hold the original values.
\end{itemize}

\subsubsection{Volatile Memory and Fileless Malware}
Modern attacks increasingly avoid writing data to the physical disk to minimize the forensic footprint left in the file system.

\subsubsection*{Memory-Only Execution}
\begin{itemize}
    \item \textbf{RAM-only Residency:} Malware resides exclusively in volatile memory (RAM), disappearing upon system reboot.
    \item \textbf{Process Injection:} Hiding malicious code within the memory space of legitimate system processes (e.g., \texttt{svchost.exe} or \texttt{explorer.exe}).
    \item \textbf{Countermeasure:} \textbf{Live Memory Acquisition} is critical. If the system is powered off before an image of the RAM is taken, the evidence is permanently lost.
\end{itemize}

\subsubsection{Detection of Anti-forensic Activity}
The presence of anti-forensic tools is itself a significant lead.
\begin{itemize}
    \item \textbf{Tool Artifacts:} The installation or execution traces of wiping software, encryption suites (e.g., VeraCrypt), or CCleaner provide circumstantial evidence of intent to conceal data.
    \item \textbf{Statistical Anomalies:} High entropy levels in files or partitions often indicate the presence of encrypted data or compressed malicious payloads.
\end{itemize}

\noindent


\subsubsection*{The "Arms Race" Paradox}
Anti-forensics is characterized by a continuous evolution: as forensic tools become more sophisticated in identifying hidden traces, anti-forensic methods shift toward more abstract layers of the system (e.g., BIOS/UEFI level rootkits or GPU-based malware), requiring investigators to maintain a multi-layered acquisition strategy.
